## Collaboration Guidelines

1. **Git Workflow:**
   - Create feature branches for each task
   - Use descriptive commit messages
   - Pull requests for code review before merging to main

2. **File Naming:**
   - Use kebab-case for HTML files
   - Use camelCase for JavaScript files
   - Use kebab-case for CSS files

3. **Code Standards:**
   - Include comments explaining your code
   - Follow consistent indentation (2 spaces)
   - Test your changes locally before pushing

4. **Communication:**
   - Update project progress in `docs/project-plan.md`
   - Report issues or blockers immediately
   - Coordinate on shared components (navbar, footer)

5. **Firebase Integration:**
   - Kyle will handle Firebase setup and configuration
   - All team members should coordinate with Kyle for database structure
   - Test Firebase features in development before production
